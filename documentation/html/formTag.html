Balises Formulaire

Form

La balise <form> définit un formulaire HTML
Elle permet à l'utilisateur d'entrer des données et de les valider pour les envoyer par exemple vers un serveur.
Le formulaire contient des éléments de formulaire comme les balises input, output, label, select...






Et en code:

  <form>

   <label for="fname">First name:</label>

   <input type="text" id="fname" name="fname" value="John">

   <label for="lname">Last name:</label>

   <input type="text" id="lname" name="lname" value="Doe">

   <input type="submit" value="Submit">

  </form>

Les inputs

La balise <input> spécifie un champ de saisie où l'utilisateur peut saisir des données.
Ces éléments sont utilisés dans un élément <form> pour déclarer des contrôles d'entrée qui permettent aux utilisateurs de saisir des données.
Un champ de saisie peut varier de plusieurs façons, selon l'attribut type.
Remarque: L'élément <input> est vide, il contient uniquement des attributs.





Et en code:

<form>


 <label>Un champ de type text:</label><input type="text" placeholder="Exemple">


 <label>Un champ de type checkbox:</label><input type="checkbox" placeholder="Exemple">


 <label>Un champ de type color:</label><input type="color" placeholder="Exemple">


 <label>Un champ de type search:</label><input type="search" placeholder="Exemple">


 <label>Un champ de type password:</label><input type="password" placeholder="Exemple">


</form>

Output

La balise <output> spécifie un champ où le résultat d'un calcul est renvoyé

0 100 + =

Et en code:

<form>

 <input type="range" id="a" value="50">100

+<input type="number" id="b" value="50">

=<output name="x" for="a b"></output>

</form>

On constate dans le code que les balises input d'id "a" et "b" sont liées à l'output par "for="a b"

Par défaut, le css du output est output {
display: inline;
}

L'attribut du label doit être lié à l'input auquel il se rapporte en utilisant un for qui correspond à l'id de l'input.

Fieldset et Legend

A quoi ça sert un fieldset et un legend?
Regardez l'exemple ci-dessous, il a été fait sans CSS. Nice, non?
Le fieldset se présente par défaut comme un cadre, donc visuellement il découpe un form en plus petites parties.
Le legend s'ajoute "dans" le cadre du fieldset par défaut, ce qui fait qu'un legend sans fieldset ne sert à rien.

Colour

Et en code:

<form>

 <fieldset>

    <legend>Colour</legend>

    <input type="radio" name="colour" value="red" id="colour_red">

    <label for="colour_red">Red </label>

    <input type="radio" name="colour" value="green" id="colour_green">

    <label for="colour_green">Green >/label>

    <input type="radio" name="colour" value="blue" id="colour_blue">

    <label for="colour_blue">Red </label>

    </fieldset>

  </form>

Textarea

La balise <textarea> définit un contrôle de saisie de texte sur plusieurs lignes.
C'est un input, mais sur plusieurs lignes et qui peut être redimensionné par l'utilisateur. On peut choisir la taille de cette zone de texte en utilisant "rows" et "cols" comme on peut le voir dans l'exemple ci-dessous. "Rows" détermine le nombre de lignes qu'occupera la zone de texte, "cols" détermine la largeur de la zone en largeur moyenne de caractères. Par défaut, la valeur cols est à 20.

Et en code:

<form>
  <label for="w3mission">Ce textarea est tout simplement magnifique:</label>
  <textarea id="w3mission" rows="4" cols="50">
    Il est très clairement magnifique, et on peut même ajouter du texte dedans.
  </textarea>
</form>

Si vous voulez en savoir plus: l'article sur textarea de w3schools.

Select, option, optgroup et boutons radio

La balise <select> définit un menu déroulant.
Option est la liste des choix dans le menu.
Optgroup est une séparation entre les choix.

Attention à ne pas confondre la balise select/option et les boutons radios.
Les boutons radios permettent de choisir un choix dans une liste. Par défaut ces boutons apparaissent sous la forme d'un rond. Il s'agit d'un type particulier d'input, le type "radio".

Voici des boutons radios, sélectionnez un parfum préféré:

Vous voyez dans le code ci-dessous que pour qu'un choix soit coché de base dans un bouton radio, il faut utiliser l'attribut checked.

<form>
 <p>Voici des boutons radios, sélectionner un parfum préféré:</p>
 <div>
 <input type="radio" id="huey" name="drone" value="huey" checked>
 <label for="huey">Vanille</label>
 </div>  <div>
 <input type="radio" id="dewey" name="drone" value="dewey">
 <label for="dewey">Chocolat</label>
 </div>
</form>

Et en code:

<form>

  <label for="cars">Choose a car:</label>

  <select id="cars">

  <optgroup label="Swedish Cars">

   <option value="volvo">Volvo</option>

   <option value="saab">;Saab</option>

  </optgroup>

  <optgroup label="German Cars">

   <option value="mercedes" selected>Mercedes</option>

   <option value="audi"> Audi</option>

  </optgroup>

 </select>

</form>

Si on veut qu'un choix soit sélectionné de base dans le menu, il suffit d'ajouter l'attribut selected à l'option.
Si on veut laisser l'utilisateur choisir plusieurs réponses, il suffit d'ajouter l'attribut multiple. Ex: <select id="cars" name="cars" multiple>

Si vous voulez en savoir plus: l'article sur select et celui sur l'article sur optgroup de w3schools.
