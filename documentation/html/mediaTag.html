Les balises multimédia

Les balises multimédia permettent de lire du contenu audio et vidéo sur le navigateur et une d’elle, donne le droit à l’utilisateur de dessiner des motifs et des formes sur une zone spécifique du navigateur.

Liste des balises media :

<Audio> : Cette balise permet l’affichage et la prise en charge de contenu audio, sous différents formats.

Pour spécifier la piste audio, on peut utiliser l’attribut src ou la balise source. L’attribut controls met à disposition de l’utilisateur une interface pour la lecture du média.

Quelques formats audio : mp3, ogg, wav, FLAC...

Quelques attributs utiles : autoplay, loop, muted, preload

    <audio controls src="/media/examples/t-rex-roar.mp3">
            Your browser does not support the audio element. 
    </audio>

<Video> : Permet de lire du contenu vidéo sous plusieurs formats. Comme la balise audio, l’attribut src et la balise source.

De même, L’attribut controls peut être spécifié pour disposer de l’interface de contrôle.

Formats vidéo : mp4, avi, mov, rm, mkv, divx...

Attributs utiles : autoplay, loop, muted, preload 

    <video controls width="250">

        <source src="https://assets.mixkit.co/videos/preview/mixkit-times-square-during-a-rainy-night-4332-large.mp4" type="video/webm">

        <source src="https://assets.mixkit.co/videos/preview/mixkit-times-square-during-a-rainy-night-4332-large.mp4" type="video/mp4">

        Sorry, your browser doesn't support embedded videos.
    </video>

<Source> (balise orpheline) : La balise source sert à définir une ou plusieurs ressources média pour sa lecture (s’accompagne avec la balise audio ou vidéo)

Quelques attributs utiles : media, sizes, src, srcset, type 

        <video> controls width="400" height="400" muted> 

        <source src="https://assets.mixkit.co/videos/preview/mixkit-stars-in-space-1610-large.mp4" type="video/mp4">
        <source src="https://assets.mixkit.co/videos/preview/mixkit-stars-in-space-1610-large.webm" type="video/webm">

            This browser does not support the HTML5 video element.
        </video>

<Embed> (balise orpheline) : La balise embed permet d'intégrer un média ou une application sur la page HTML.

Quelques attributs qui l’accompagne (conseillé de toutes les définir et spécifier): height, src, type, width

        <embed width="560" height="315" src="https://www.youtube.com/embed/9jBL_2UiRw8" frameborder="0" allow="accelerometer; autoplay; encrypted-media; gyroscope; picture-in-picture" allowfullscreen></embed>

<Picture>: Cette balise agit comme les balises audio et vidéo mais pour les images.

Petit point a apporter : La prise en charge s'applique aussi bien pour les images matricielles que vectorielles

Differences entre les 2 types ? => Une image vectorielle peut être agrandie sans perdre sa qualité alors qu'une image matricielle perd en netteté à l'agrandissement. L'image matricielle (ou bitmap): Elle est composée de petits points appelés « pixels » que l'on ne voit pas à l'œil nu.

Image matricielle (ou bitmap) : Lors de l’agrandissement d’une image matricielle, cette dernière devient floue car les pixels ressortent, ce sont les carrés qui apparaissent sur l’écran.

Image vectorielle : Elle est composée de lignes de segments qui sont liés entre eux par des formules mathématiques. Il s’agit d’un système de proportionnalité et de coordonnées. Grâce à la vectorisation, chaque élément a une place bien définie ce qui empêche la déformation de l’image.

Quelques attributs utiles : media, sizes, src, srcset, type

        <picture>
            <source srcset="https://thumbs.gfycat.com/SkeletalBeneficialGrebe-size_restricted.gif" media="(min-width: 800px)">
            <img src="https://thumbs.gfycat.com/SkeletalBeneficialGrebe-size_restricted.gif"/>
        <picture>

<img> (balise orpheline) : Affiche une image.

Petite précision CSS : Les images sont définies en mode display: inline par défaut

Format d’images prise en charge : jpeg, png, svg, tiff, gif, webP, bmp…

Attributs pris en charge : alt, crossorigin, sync, async, ismap, size, src, srcset, width..

      <img class="dark-vador" src="https://vignette.wikia.nocookie.net/fr.starwars/images/c/cd/Star_Wars_Dark_Vador_Vador.jpg/revision/latest?cb=20180204092550" alt="Dark Vador Marvel">

<object> permet d'insérer une ressource de type media, application ou plugin.

Petite précision : Les nouveaux navigateurs ne sont ne prennent plus en charge (ou partiellement demandant autorisation à l'utilisateur). Adobe Flash, ActiveX, les applets et plugins Java

Attributs pris en charge : data, form, width, height, type, name... 

      <object type="application/pdf" data="https://gallica.bnf.fr/ark:/12148/bpt6k201333r.pdf" width="400" height="400"></object>

Attributs : value, name

<track> permet d’ajouter des sous-titres dans un média de type audio et vidéo.

Attributs : default, kind (subtitles, captions, descriptions, chapters, metadata), label, src, srclang

Précision sur les formats de sous-titres : Ces derniers sont généralement au format .vtt (Web Video Text Tracks File). 

      
    <video controls width="250" src="/media/examples/friday.mp4">
    <track default kind="captions" srclang="en" src="/media/examples/friday.vtt"/>

    Sorry, your browser doesn't support embedded videos.

<canvas>

Cette balise permet de créer une zone sur le navigateur sur lequel l’utilisateur peut injecter du code javascript (via l’API Canvas) pour créer des animations, faire du dessin SVG et des graphes, voir même de la 3D(via l’API webGL).

        <script type="application/javascript"> 
            function draw() {
              var canvas = document.getElementById("canvas");
              if (canvas.getContext) {
                var ctx = canvas.getContext("2d");
        
                ctx.fillStyle = 'rgb(200, 0, 0)';
                ctx.fillRect(10, 10, 50, 50);
        
                ctx.fillStyle = 'rgba(0, 0, 200, 0.5)';
                ctx.fillRect(30, 30, 50, 50);
              }
            }
          </script>
         <body onload="draw();">
           <canvas id="canvas" width="400" height="400"></canvas>
         </body>

<svg>

La balise <SVG> permet d'insérer du code SVG de définir des coordonnées en 2 dimensions pour reproduire des formes (polygon, rectangle, cercle, sphère...), des dessins et même des animations.

    <svg xmlns="http://www.w3.org/2000/svg" 
    width="150" height="100" viewBox="0 0 3 2">
        <rect width="1" height="2" x="0" fill="#008d46" />
        <rect width="1" height="2" x="1" fill="#ffffff" />
        <rect width="1" height="2" x="2" fill="#d2232c" />
    </svg>
